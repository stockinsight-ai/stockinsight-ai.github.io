Claude Codeで株式分析システムを完全自動実装｜AIとペアプログラミングした開発体験記

公開日: 2026年4月30日

StockInsight AIはAIが実装したシステムです

StockInsight AIの銘柄選定システムは、Anthropic社のClaude CodeというAIアシスタントとのペアプログラミングによって開発されました。

「AIが株を選ぶ」というサービスですが、実はそのシステム自体もAIが設計・実装しています。この記事では、どのようにAIと協力して開発を進めたか、その体験をリアルに紹介します。

Claude Codeとは

Claude Codeは、Anthropic社が提供するコーディング特化のAIアシスタントです。ターミナル上で動作し、コードの読み書き・テスト実行・リファクタリングなど、ソフトウェア開発のほぼすべての工程を自律的にこなします。

# このようにターミナルから起動して使う
claude

「何を作りたいか」を自然言語で伝えると、ファイルを作成し、コードを書き、テストを実行し、エラーを修正してくれます。

開発の全体フロー

1. 仕様策定（人間が主導）

最初にやったのは「どんな銘柄を選ぶか」という仕様の明確化です。ここは人間が主体的に考える必要がありました。

Claude Codeに対して、「バリュー投資家・グロース投資家・テクニカル分析家の3つの視点でディスカッションして、スクリーニング条件を決めてほしい」と依頼しました。

AIは3人の仮想投資家キャラクターとして以下のような議論を展開しました：

バリュー投資家A： 「ROE 8%以上、自己資本比率30%以上は最低条件。財務の健全性なくして長期投資は成立しない」

グロース投資家B： 「ROEより成長率が重要。PEG比率1.0以下で、成長に見合った価格かを確認する」

テクニカル分析家C： 「いくら良い会社でも上昇トレンドでなければ買わない。週足MACDのゴールデンクロスを確認する」

この「仮想ディスカッション」を通じて、テクニカル40点・ファンダメンタル40点・センチメント20点の3軸スコアリングという設計に落ち着きました。

2. アーキテクチャ設計（AIが提案）

「クリーンアーキテクチャでモジュールを分離する」という設計はClaude Codeが提案しました。

src/
├── data/        # データ取得（yfinance・スクレイピング）
├── analysis/    # スクリーニング・スコアリング
├── report/      # レポート生成
└── notion/      # Notion APIとの連携

データ取得・分析・出力が完全に分離された構造のおかげで、後から「Web公開機能」を追加するときも、既存コードを変更せずに src/site/ ディレクトリを追加するだけで済みました。

3. 実装（AIが主導、人間がレビュー）

コードの実装はほぼClaude Codeが担当しました。たとえばテクニカル指標の計算は：

# Claude Codeが実装したテクニカルスコア計算の核心部分
import pandas_ta as ta

def calc_technical_score(df: pd.DataFrame) -> dict:
    """週足データからテクニカルスコア（40点満点）を計算"""
    score = 0
    reasons = []

    # 25週移動平均線チェック（15点）
    ma25 = df["Close"].rolling(25).mean()
    if df["Close"].iloc[-1] > ma25.iloc[-1] and ma25.iloc[-1] > ma25.iloc[-5]:
        score += 15
        reasons.append("25週MA上抜け・上向き")

    # 週足MACDゴールデンクロス（15点）
    macd = ta.macd(df["Close"], fast=12, slow=26, signal=9)
    if (macd["MACD_12_26_9"].iloc[-1] > macd["MACDs_12_26_9"].iloc[-1] and
        macd["MACD_12_26_9"].iloc[-2] <= macd["MACDs_12_26_9"].iloc[-2]):
        score += 15
        reasons.append("週足MACDゴールデンクロス")

    # 出来高比率チェック（10点）
    vol_ratio = df["Volume"].iloc[-4:].mean() / df["Volume"].iloc[-17:-4].mean()
    if vol_ratio >= 1.3:
        score += 10
        reasons.append(f"出来高比率{vol_ratio:.2f}倍")

    return {"score": score, "reasons": reasons}

人間がやったのは「このコードは本当に意図通りか」というレビューと、「実際の株価データで動かしたときの結果は正しいか」の確認です。

4. テスト（AIが設計・実装）

Claude Codeは自動でテストコードも書きました。

tests/
├── unit/           # 各関数の単体テスト
├── integration/    # 全体フローの結合テスト
└── e2e/            # Notion登録までの疎通テスト

特に印象的だったのは、「異常系のテスト」もきちんと設計してくれたことです。「Notion APIがタイムアウトしたとき」「yfinanceが空データを返したとき」など、本番で起きうるエラーケースを網羅したテストが自動生成されました。

AIに任せて良かった点・注意が必要だった点

任せて良かった点

ボイラープレートの排除： 設定ファイル・ディレクトリ構成・import文など「考えるほどでもない定型作業」をすべてやってくれる

リファクタリングの徹底： 「コードが重複している」「この関数は長すぎる」など、人間なら見逃しがちな品質問題を即指摘・修正

ドキュメント生成： 仕様書・設計書・テスト設計書も自動生成してくれた

注意が必要だった点

投資判断はAIに任せない： どんな指標を使うか・何点以上を推奨とするかなど、ドメイン知識が必要な判断は人間が行う必要がありました

データの正確性確認： yfinanceから取得した株価データが正しいか、実際の証券サイトと照合する作業は人間が行いました

法律・規制の確認： 投資助言業登録の要否、免責事項の表現など、法的判断は必ず専門家または自分で確認が必要です

開発にかかった期間

フェーズ 所要時間 担当
仕様策定・ディスカッション 約2日 人間 + Claude Code
アーキテクチャ設計 約0.5日 Claude Code（人間レビュー）
全モジュール実装 約3日 Claude Code（人間レビュー）
テスト実装・修正 約1日 Claude Code
Web公開機能追加 約2日 Claude Code
合計 約1〜2週間

一人のエンジニアが従来の方法で同等のシステムを作るとすると、おそらく2〜3ヶ月かかる規模のプロジェクトです。それが約2週間で動くところまで到達できたのは、AIとのペアプログラミングの力です。

StockInsight AIはどのように動いているか

現在のシステムは毎週日曜日の夜に自動実行されます。

[毎週日曜 自動実行]
↓
① 東証全銘柄のスクリーニング（yfinance + 財務データ）
↓
② テクニカル・ファンダ・センチメント3軸スコア計算
↓
③ 上位10銘柄を選定（スコア60点以上）
↓
④ Notionに推奨銘柄レポートを自動登録
↓
⑤ Webサイトのデータを更新・公開

このパイプライン全体がPythonで実装されており、cronで毎週自動実行されています。

まとめ

「AIが実装したシステムが、AIを使って株を分析する」という二重の意味でAI活用が進んでいるのがStockInsight AIです。

重要なのは、AIはあくまでツールであり、投資判断や仕様策定という本質的な意思決定は人間が行うという点です。AIが得意な「大量の定型作業」「コードの品質維持」「ドキュメント生成」を任せることで、人間は「何を作るか・なぜ作るか」という価値判断に集中できます。

StockInsight AIのスコアも同様に、あくまで「参考情報」です。最終的な投資判断はご自身の知識・経験・リスク許容度に基づいて行ってください。
